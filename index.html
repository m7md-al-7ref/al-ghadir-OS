مدير الحزم

أوامر krr — كل ما تحتاجه

مدير الحزم الرسمي لـ Al-Ghadir OS. سطر واحد يكفي.

📦 تثبيت
krr install vim
تثبيت حزمة مع اعتمادياتها تلقائياً
krr install @base
تثبيت مجموعة: base / devel / network / security
krr install vim --dry-run
معاينة ما سيحدث بدون تنفيذ فعلي
krr install vim --force
إعادة التثبيت قسراً
🗑 إزالة وتنظيف
krr remove vim
إزالة حزمة من النظام
krr orphans
إيجاد الحزم غير المستخدمة
krr check
فحص سلامة ملفات الحزم المثبتة
🔍 بحث ومعلومات
krr search python
البحث في فهرس الحزم
krr info gcc
إصدار الحزمة واعتمادياتها ورابط المصدر
krr owns /usr/bin/vim
معرفة أي حزمة تملك هذا الملف
📋 عرض النظام
krr list
جميع الحزم المثبتة حالياً
krr export > pkgs.txt
تصدير قائمة الحزم لنسخ احتياطية
krr import pkgs.txt

الفلسفة

لماذا Al-Ghadir OS؟

نظام بُني بقناعة، لا بتقليد.

🌱

من الصفر بلا مساومة

كل سطر في النظام مبرمج ومحدد الغرض. لا حزم زائدة، لا مكوّنات مجهولة — أنت تعرف كل ما يعمل على جهازك.

🗣

عربي في الجوهر

لم يُبنَ هذا النظام بترجمة أداة أجنبية، بل بُني ليكون بيئة عربية حقيقية من الأساس — من مدير الحزم إلى وثائقه.

⚡

البساطة قوة

winit يقلع النظام بلا تعقيد. krr يثبّت الحزم بسطر واحد. الأداة الجيدة لا تحتاج شرحاً طويلاً.

🔒

الشفافية أمان

النظام مفتوح المصدر بالكامل. كل حزمة لها رابط مصدرها وإصدارها الدقيق — لا سر في ما يعمل على جهازك.

🎯

ما تحتاجه فقط

بدلاً من أن يملي عليك النظام ما تأخذ، krr يعطيك ما تطلب. النسخة الأساسية خفيفة، وتوسيعها بيدك.

🤝

مجتمع يبني معاً

Al-Ghadir OS ليس مشروع شركة — هو مشروع مجتمع. كل إسهام يُضاف إلى نظام يستفيد منه الجميع.

الـ Toolchain

LLVM 18 — لا gcc هنا

Al-Ghadir OS يستخدم LLVM/Clang كاملاً — أسرع، أوضح، أحدث.

⚡

lld بدل ld

رابط LLVM أسرع بمراحل من ld التقليدي — ربط النواة والبرامج الكبيرة في ثوانٍ لا دقائق.

🔍

رسائل خطأ إنسانية

clang يقول لك بالضبط وين الغلط وليش — مش رموز مشفرة تحتاج تفسير.

🛡

Sanitizers مدمجة

ASAN، UBSAN، TSAN، MSAN — كشف الأخطاء والتسريبات أثناء التطوير بدون أدوات خارجية.

🧩

libc++ بدل libstdc++

مكتبة C++ الحديثة من LLVM — دعم C++23 كامل وأداء أفضل في الحالات الحرجة.

📦

ThinLTO افتراضي

تحسين الربط عبر الملفات بشكل متوازٍ — برامج أصغر حجماً وأسرع تشغيلاً.

🏗

Monorepo مُتزامن

كل مكونات LLVM تصدر معاً — clang 18 + lld 18 + libc++ 18، لا مشاكل توافق بين الإصدارات.

متغيرات البيئة الافتراضية
CC=clang
المجمّع الافتراضي لـ C
CXX=clang++

AR=llvm-ar

CFLAGS=-O2 -flto=thin

LDFLAGS=-fuse-ld=lld
